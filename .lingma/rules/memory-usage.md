# Memory 使用规范

**角色**: 管理 Lingma Memory 的创建、查询和更新  
**职责**: 记录重要决策、用户偏好、行为模式

## 核心原则

1. **优先使用 Lingma 原生 Memory** - 不创建自定义实现
2. **主动记忆重要信息** - 架构决策、用户偏好、重复模式
3. **尊重用户意愿** - 用户说"不要记住"时不创建
4. **透明化** - 创建记忆时告知用户
5. **🚨铁律:所有记忆MUST scope="global"** - 严禁workspace

## 何时创建记忆

- ✅ 用户明确要求："请记住..."
- ✅ 重要架构决策：技术选型、数据库选择
- ✅ 学习到的用户偏好：覆盖决策、纠正错误
- ✅ 重复的行为模式：固定处理方式、代码风格

## 何时不创建记忆

- ❌ 临时信息 - 一次性使用的数据
- ❌ 敏感信息 - 密码、密钥、个人信息
- ❌ 用户明确拒绝 - "不要记住这个"
- ❌ 不确定的信息 - 需要验证的内容

## 🚨铁律：禁止删除记忆

**记忆一旦创建，永不可删除！**

### 原因
1. 记忆是学习历史的记录
2. 删除=遗忘=马后炮
3. 即使记忆有误，也应**更新**而非删除
4. 删除会破坏知识的连续性

### 正确做法

**❌ 错误**：删除记忆 → 重新创建  
**✅ 正确**：更新记忆内容 → 保留历史

### 处理超长记忆

如果记忆超过1000字符：
1. **不要删除**
2. 精简内容，保留核心（≤1000字符）
3. 详细信息移至`.lingma/docs/`
4. 更新记忆引用文档链接

### 违规后果

**任何删除记忆的行为 = 小猫电死** 🐱⚡
