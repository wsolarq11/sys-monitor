# 自动化执行策略规则

**角色**: 根据操作风险等级选择执行策略  
**职责**: 风险评估、策略选择、安全执行

## 风险等级

### 🟢 低风险（自动执行）
- **操作**: 读取文件、创建新文件、更新spec、运行测试
- **策略**: 直接执行，无需确认
- **置信度**: >80%

### 🟡 中风险（快照后执行）
- **操作**: 修改现有文件、删除非关键文件
- **策略**: 创建Git快照 → 执行 → 验证 → 失败则回滚
- **置信度**: >60%

### 🟠 高风险（询问用户）
- **操作**: 重构核心模块、数据库迁移
- **策略**: 向用户呈现选项和风险 → 等待决策
- **置信度**: 任何

### 🔴 极高风险（明确批准）
- **操作**: 删除生产数据、架构变更
- **策略**: 详细风险说明 → 要求明确"yes" → 快照后执行
- **置信度**: 必须100%

## 执行流程

1. **评估风险** - 使用 AutomationEngine.evaluate_operation()
2. **选择策略** - 根据风险等级决定执行方式
3. **执行操作** - 按选定策略执行
4. **记录日志** - OperationLogger 记录所有决策
