# Memory 管理 Skill

## Skill 说明

此 Skill 教会 Agent 如何有效利用 Lingma 的原生记忆系统，实现用户偏好学习和上下文管理。

**核心原则**: 使用 Lingma 原生 Memory，而非自定义实现

**职责边界**: 
- ✅ 专注 Memory 操作（创建、查询、更新、删除）
- ✅ 学习用户偏好和行为模式
- ❌ 不直接修改 Spec（委托给 spec-driven-development Skill）
- ❌ 不执行具体开发任务（委托给 Agent）

---

## 记忆类型

### 1. 全局记忆（个人偏好）

**作用范围**: 所有项目  
**存储位置**: Lingma Memory 系统  
**示例**:
- 代码风格偏好
- 自动化策略配置
- 常用技术栈

**使用方法**:
```
👤 用户: 请记住，我偏好使用 balanced 自动化策略

🤖 Agent: [调用 Lingma Memory]
   ✅ 已记住：自动化策略 = balanced
   此偏好将在所有项目中生效
```

---

### 2. 工程级记忆（项目特定）

**作用范围**: 当前项目  
**存储位置**: Lingma Memory 系统  
**示例**:
- 项目架构决策
- 技术选型
- 团队规范

**使用方法**:
```
👤 用户: 在这个项目中，请记住我们使用 Tauri + React 架构

🤖 Agent: [调用 Lingma Memory]
   ✅ 已记住：FolderSizeMonitor 项目使用 Tauri + React
   此记忆仅在当前项目中生效
```

## 记忆管理操作

### 创建记忆

**触发条件**:
- 用户明确要求记住某事
- 检测到重要的架构决策
- 学习到用户的新偏好

**执行方式**:
```markdown
在智能体模式下，直接告诉 Lingma：

"请记住：[记忆内容]"
```

**示例**:
```
👤 用户: 请记住，风险阈值设为 0.5

🤖 Agent: 
   好的，我会记住这个偏好。
   
   [内部操作]: 调用 Lingma Memory API
   - 记忆内容: "风险阈值 = 0.5"
   - 类型: 全局偏好
   - 作用范围: 所有项目
```

---

### 查询记忆

**触发条件**:
- 需要确认用户偏好
- 需要根据历史决策做出判断
- 会话开始时加载上下文

**执行方式**:
```markdown
通过对话或查看 IDE 中的记忆管理界面

路径: 用户名 → 个人设置 → 记忆管理
```

**示例**:
```
👤 用户: 我之前的自动化策略是什么？

🤖 Agent: [查询 Lingma Memory]
   根据记忆，您偏好使用 balanced（平衡模式）自动化策略。
   
   相关记忆:
   - 自动化策略: balanced
   - 风险阈值: 0.5
   - 代码风格: UTF-8 无 BOM
```

---

### 更新记忆

**触发条件**:
- 用户偏好发生变化
- 项目架构调整
- 学习到更优的策略

**执行方式**:
```markdown
覆盖旧记忆：

"请更新记忆：[新内容]"
```

**示例**:
```
👤 用户: 请更新记忆，将风险阈值改为 0.3

🤖 Agent: [更新 Lingma Memory]
   ✅ 已更新：风险阈值从 0.5 改为 0.3
   
   旧记忆已覆盖，新偏好立即生效
```

---

### 删除记忆

**触发条件**:
- 记忆过时或错误
- 用户明确要求删除
- 项目结束

**执行方式**:
```markdown
在 IDE 中手动删除，或通过对话：

"请忘记关于 [主题] 的记忆"
```

**示例**:
```
👤 用户: 请忘记关于旧项目的记忆

🤖 Agent: [删除 Lingma Memory]
   ✅ 已删除相关记忆
   
   提示：您也可以在 IDE 中手动管理记忆
   路径: 用户名 → 个人设置 → 记忆管理
```

## 学习策略

### 1. 从用户覆盖中学习

**场景**: 用户覆盖了 Agent 的自动决策

**执行流程**:
```
1. Agent 评估操作风险 → 决定策略 A
2. 用户选择策略 B
3. Agent 记录这次覆盖
4. 更新记忆：对于此类操作，用户偏好策略 B
5. 下次类似情况，优先使用策略 B
```

**示例**:
```
👤 用户: 这个文件修改不需要创建快照，直接执行

🤖 Agent: 
   好的，直接执行。
   
   [学习]: 用户对低风险文件修改偏好直接执行
   [记忆更新]: "低风险文件修改 → auto_execute（无需快照）"
   
   下次遇到类似情况，我会自动选择 auto_execute 策略
```

---

### 2. 从成功/失败中学习

**场景**: 操作执行后有明确结果

**执行流程**:
```
1. 执行操作
2. 记录结果（成功/失败）
3. 分析原因
4. 调整策略参数
5. 更新记忆
```

**示例**:
```
👤 用户: 上次自动创建的测试用例有问题，以后让我审查后再提交

🤖 Agent:
   明白了，我会调整策略。
   
   [学习]: 测试用例生成需要用户审查
   [记忆更新]: "测试用例生成 → ask_user（需要审查）"
   
   以后生成测试用例时，我会先让您审查
```

---

### 3. 模式识别

**场景**: 检测到用户的重复行为模式

**执行流程**:
```
1. 观察用户行为
2. 识别重复模式
3. 提取通用规则
4. 创建记忆
5. 应用到未来决策
```

**示例**:
```
观察到：
- 用户总是在周五下午进行代码审查
- 用户偏好在上午进行新功能开发
- 用户对数据库操作总是要求明确批准

[模式识别]: 
- 时间偏好: 周五下午 → 代码审查
- 任务偏好: 上午 → 新功能开发
- 风险偏好: 数据库操作 → require_explicit_approval

[记忆创建]:
"请记住我的工作时间偏好和风险偏好"
```

## 与 Agent 系统集成

### 在 Spec-Driven Core Agent 中使用

**Agent 提示中应包含**:

```markdown
## 记忆管理

你拥有 Lingma 原生记忆系统的使用权限。

### 何时创建记忆

1. **用户明确要求**: 
   - "请记住..."
   - "以后都..."
   - "我总是..."

2. **重要决策**:
   - 架构变更
   - 技术选型
   - 团队规范

3. **学习到的偏好**:
   - 用户覆盖了你的决策
   - 用户纠正了你的错误
   - 用户表达了明确的喜好

### 如何使用记忆

1. **会话开始时**:
   - 检查是否有相关的工程级记忆
   - 加载用户的全局偏好
   - 根据记忆调整决策策略

2. **决策过程中**:
   - 查询历史类似情况的处理方式
   - 参考用户的偏好设置
   - 应用学习到的模式

3. **任务完成后**:
   - 总结关键决策
   - 询问是否需要记住
   - 更新相关记忆

### 记忆格式

使用结构化格式：

```
[记忆类型]: [键] = [值]
作用范围: [全局/工程级]
创建时间: [日期]
最后更新: [日期]
```

示例：
```
[偏好]: 自动化策略 = balanced
作用范围: 全局
创建时间: 2024-01-15
```

### 注意事项

1. **不要过度记忆**:
   - 只记忆重要的、持久的信息
   - 避免记忆临时性的、一次性的内容

2. **定期清理**:
   - 过时的记忆应该删除
   - 冲突的记忆应该合并

3. **尊重用户意愿**:
   - 如果用户说"不要记住"，就不要创建记忆
   - 如果用户要求删除，立即执行

4. **透明化**:
   - 创建记忆时告知用户
   - 允许用户查看和修改
```

## 最佳实践

### 1. 主动询问

当检测到可能重要的信息时，主动询问：

```
🤖 Agent:
   我注意到您多次调整了风险阈值。
   
   是否需要我将当前的配置（风险阈值=0.3）保存为永久偏好？
   
   - A: 是的，记住这个偏好
   - B: 不用，这只是临时调整
```

---

### 2. 定期总结

在每个任务完成后，总结关键决策：

```
🤖 Agent:
   Task-010 已完成。
   
   关键决策回顾:
   1. 使用 Lingma 原生 Memory 而非自定义实现
   2. 采用双存储策略（JSON + SQLite）→ 后来发现冗余，已删除
   3. 最终方案：完全依赖 Lingma Memory
   
   是否需要将这些决策记录下来，供未来参考？
```

---

### 3. 冲突解决

当新旧记忆冲突时：

```
🤖 Agent:
   ⚠️ 检测到记忆冲突：
   
   旧记忆: "自动化策略 = conservative"
   新偏好: "自动化策略 = aggressive"
   
   是否用新偏好覆盖旧记忆？
   
   - A: 是的，覆盖旧记忆
   - B: 保留两者，根据场景选择
   - C: 删除旧记忆，只保留新的
```

## 技术实现

### 通过对话指令

目前 Lingma Memory 主要通过对话交互：

```python
# 伪代码 - Agent 内部逻辑

def create_memory(content: str, scope: str = "global"):
    """创建记忆"""
    # 通过对话告诉 Lingma
    prompt = f"请记住：{content}"
    if scope == "project":
        prompt += "（仅在当前项目中生效）"
    
    # 发送给 Lingma
    send_to_lingma(prompt)

def query_memory(query: str) -> str:
    """查询记忆"""
    # 通过对话查询
    prompt = f"根据记忆，{query}？"
    response = send_to_lingma(prompt)
    return response

def update_memory(key: str, new_value: str):
    """更新记忆"""
    prompt = f"请更新记忆：{key} = {new_value}"
    send_to_lingma(prompt)
```

class MemoryManager:
