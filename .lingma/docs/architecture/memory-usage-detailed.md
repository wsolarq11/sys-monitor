## 📋 完整使用指南

### 何时创建记忆

#### 1. 用户明确要求

当用户说：
- "请记住..."
- "以后都..."
- "我总是..."
- "记住这个偏好"

**示例**:
```
👤 用户: 请记住，我偏好使用 balanced 自动化策略

✅ Agent: 已记住您的偏好
   [内部]: 调用 Lingma Memory API
   - 内容: "自动化策略 = balanced"
   - 类型: 全局偏好
```

#### 2. 重要的架构决策

涉及：
- 技术选型
- 架构模式
- 数据库选择
- 框架版本

**示例**:
```
👤 用户: 我们决定使用 SQLite 而不是 PostgreSQL

✅ Agent: 这是一个重要的架构决策
   [内部]: 创建工程级记忆
   - 内容: "FolderSizeMonitor 项目使用 SQLite"
   - 类型: 工程级记忆
   - 作用范围: 当前项目
```

#### 3. 学习到的用户偏好

检测到：
- 用户覆盖了你的决策
- 用户纠正了你的错误
- 用户表达了明确的喜好

#### 4. 重复的行为模式

观察到：
- 用户在特定时间做特定任务
- 用户对某类操作有固定处理方式
- 用户的代码风格偏好

---

### ❌ 不应该创建记忆的情况

1. **临时信息** - 一次性使用的数据
2. **敏感信息** - 密码、密钥、个人信息
3. **用户明确拒绝** - "不要记住这个"
4. **不确定的信息** - 需要验证的内容
5. **过于细节** - 不影响长期工作的细节

## 🔧 Memory 类型

### 1. 全局偏好 (Global Preferences)

**作用范围**: 所有项目  
**示例**: 
- 编码风格偏好
- 自动执行策略偏好
- 沟通方式偏好

### 2. 工程级记忆 (Project-Level)

**作用范围**: 当前项目  
**示例**:
- 技术栈选择
- 架构决策
- 项目规范

### 3. 会话级记忆 (Session-Level)

**作用范围**: 当前会话  
**示例**:
- 临时上下文
- 正在进行的工作
- 短期目标

---

## 📝 记忆格式

### 标准格式

```markdown
**类型**: [全局偏好/工程级/会话级]  
**内容**: [简洁描述]  
**作用范围**: [all/project/session]  
**创建时间**: YYYY-MM-DD  
**来源**: [用户明确要求/观察到/学习到]
```

### 示例

```markdown
**类型**: 全局偏好  
**内容**: 自动化策略 = balanced  
**作用范围**: all  
**创建时间**: 2026-04-15  
**来源**: 用户明确要求
```

## 🛠️ 使用 Lingma Memory API

### 创建记忆

```python
# 伪代码
lingma.memory.create({
    "content": "自动化策略 = balanced",
    "type": "global_preference",
    "scope": "all"
})
```

### 查询记忆

```python
# 伪代码
memories = lingma.memory.query({
    "type": "global_preference",
    "scope": "all"
})
```

### 更新记忆

```python
# 伪代码
lingma.memory.update({
    "id": "memory_id",
    "content": "自动化策略 = aggressive"
})
```

### 删除记忆

```python
# 伪代码
lingma.memory.delete("memory_id")
```

## 💡 最佳实践

### 1. 透明化

创建记忆时告知用户：
```
✅ 我已记住您的偏好：自动化策略 = balanced
   （这将影响未来的自动化决策）
```

### 2. 尊重用户意愿

用户说"不要记住"时：
```
✅ 好的，我不会记住这次的选择
```

### 3. 定期回顾

每月检查记忆列表：
- 删除过时的记忆
- 更新不准确的信息
- 合并相似的条目

### 4. 优先级管理

- P0: 用户明确要求的偏好
- P1: 重要的架构决策
- P2: 学习到的行为模式
- P3: 观察到的重复模式

## ⚠️ 常见问题

### Q1: 记忆太多怎么办？

**解决**:
1. 定期清理（每月一次）
2. 合并相似条目
3. 删除过时信息
4. 按优先级排序

### Q2: 记忆冲突怎么办？

**解决**:
1. 优先使用最新的记忆
2. 用户明确要求的优先
3. 询问用户哪个是正确的
4. 记录冲突并标记待解决

### Q3: 如何验证记忆是否生效？

**解决**:
1. 在新会话中测试
2. 观察 Agent 行为是否符合记忆
3. 如果不符，检查记忆是否正确加载
4. 必要时重新创建记忆
