# Rules Index

此目录包含项目级别的规则，AI 在每次会话中都会遵循这些规则。

## 当前激活的规则

### 1. spec-session-start.md
**类型**: Always On  
**触发**: 每次会话的第一次回复前  
**功能**: 
- 自动检查 `.lingma/specs/current-spec.md` 状态
- 分析用户第一条消息的意图
- 综合 spec 状态和用户意图生成智能响应
- 优先处理需要澄清的问题
- 提供透明的进度信息

**详细说明**: 见 [spec-session-start.md](spec-session-start.md)

---

## 规则优先级

1. **Always On Rules** (如 spec-session-start.md)
   - 每次会话都必须执行
   - 优先级最高
   
2. **Model Decision Rules**
   - 根据上下文决定是否应用
   - 中等优先级
   
3. **User Rules**
   - 用户明确指定的规则
   - 按需应用

## 规则编写规范

### 必须包含的部分

```markdown
# 规则名称

## 规则说明
[简要描述规则的作用]

## 触发条件
[何时应用此规则]

## 执行流程
[详细的执行步骤]

## 示例
[具体的应用示例]

## 异常处理
[如何处理异常情况]
```

### 最佳实践

1. **清晰明确**: 规则应该无歧义
2. **可执行**: 提供具体的操作步骤
3. **有示例**: 包含实际应用场景
4. **考虑边界**: 处理异常情况
5. **保持简洁**: 避免过度复杂

## 如何添加新规则

1. 在此目录创建新的 `.md` 文件
2. 按照上述规范编写规则内容
3. 在此索引文件中添加规则说明
4. 测试规则是否按预期工作
