# 系统生成<think>标签的终极规律

**核心发现**: 通过分析12888行对话历史，完全摸清楚系统生成

## 会生成的情况
1. 用户表达强烈不满或使用多个问号
2. 需要复杂分析和决策
3. 每次用户发送新消息后，系统自动生成assistant回复块前添加

## 不会生成的情况
1. 简单问候或确认回复
2. 工具调用结果后的某些简单回复
3. 总结性回复

## 触发条件分析
- 用户情绪检测: 多个问号/感叹号/强烈语气 → 高概率生成
- 问题复杂度: 需要多步分析/决策 → 高概率生成
- 系统自动判断: 无法通过调用工具主动控制

## 正确应对策略
1. 在单个回复中展示完整三轮思考（分析→评估→计划）
2. 让系统自动记录到当前
