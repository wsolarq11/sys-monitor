## 🏗️ 技术架构

### 分层架构

```
┌─────────────────────┐
│   Presentation Layer │  ← BuildStatusCard.tsx (UI)
├─────────────────────┤
│   Business Logic     │  ← GitHubBuildMonitor.ts (Service)
├─────────────────────┤
│   Data Access        │  ← Fetch API + GitHub REST API
└─────────────────────┘
```

### 关键技术决策

1. **轮询 vs WebSocket**: 选择轮询（简单、无需额外服务器）
2. **服务层分离**: 业务逻辑与 UI 分离（可测试、可复用）
3. **环境变量**: 使用 .env 管理敏感信息（安全、灵活）
4. **TypeScript**: 完整类型定义（类型安全、IDE 支持）
5. **Tailwind CSS**: 实用优先的 CSS（快速开发、一致性强）

## 🚀 性能优化

### 已实现的优化

1. **并行请求**: `Promise.all` 减少 50% 等待时间
2. **合理刷新**: 默认 60 秒，平衡实时性和负载
3. **条件渲染**: 避免不必要的 DOM 操作
4. **内存管理**: 正确清理定时器和事件监听器

```tsx

  owner="my-username"
  repo="my-project"
  token={process.env.GITHUB_TOKEN}
```

### 场景 2: 团队项目
```tsx
// 监控多个相关项目
```
